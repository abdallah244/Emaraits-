## 🚀 كيف تساهم

1. **Fork** المستودع
2. أنشئ فرع جديد من `main`:
   ```bash
   git checkout -b feature/اسم-الميزة
   ```
3. اعمل التعديلات المطلوبة
4. تأكد من اجتياز جميع الاختبارات
5. ارفع التعديلات:
   ```bash
   git push origin feature/اسم-الميزة
   ```
6. افتح **Pull Request**

## ⚙️ إعداد بيئة التطوير

### المتطلبات الأساسية

- Node.js >= 18.x
- PostgreSQL >= 14
- Flutter SDK >= 3.x
- Git

### خطوات الإعداد

```bash
# استنساخ المستودع
git clone https://github.com/your-username/emaraits.git
cd emaraits

# إعداد الباك إند
cd backend
cp .env.example .env
npm install
npm run db:migrate
npm run dev

# إعداد لوحة التحكم
cd ../dashboard
cp .env.example .env
npm install
npm run dev

# إعداد تطبيق الموبايل
cd ../mobile
cp .env.example .env
flutter pub get
flutter run
```

## 📏 معايير الكود

### عام

- استخدم **TypeScript** في الباك إند ولوحة التحكم
- استخدم **Dart** في تطبيق الموبايل
- التزم بقواعد **ESLint** و **Prettier** المُعدّة في المشروع
- اكتب اختبارات لأي كود جديد

### تسمية الملفات

| النوع                 | الصيغة     | مثال             |
| --------------------- | ---------- | ---------------- |
| المكونات (Components) | PascalCase | `ClientCard.tsx` |
| الخدمات (Services)    | camelCase  | `authService.ts` |
| النماذج (Models)      | PascalCase | `Client.ts`      |
| الأدوات (Utils)       | camelCase  | `formatDate.ts`  |

### هيكل الكود

- فصل منطق العمل (Business Logic) عن طبقة العرض (UI)
- استخدام الـ Services للتعامل مع الـ API
- تجنب تكرار الكود (DRY Principle)
- كتابة تعليقات واضحة للأجزاء المعقدة

## 📝 اتفاقية الكوميتات

نتبع معيار [Conventional Commits](https://www.conventionalcommits.org/):

```
<النوع>(<النطاق>): <الوصف>
```

### أنواع الكوميتات

| النوع      | الاستخدام                                   |
| ---------- | ------------------------------------------- |
| `feat`     | ميزة جديدة                                  |
| `fix`      | إصلاح خطأ                                   |
| `docs`     | تغييرات في التوثيق                          |
| `style`    | تغييرات في تنسيق الكود (لا تؤثر على المنطق) |
| `refactor` | إعادة هيكلة الكود                           |
| `test`     | إضافة أو تحديث اختبارات                     |
| `chore`    | تغييرات في أدوات البناء أو المكتبات         |
| `perf`     | تحسينات في الأداء                           |

### أمثلة

```bash
feat(clients): إضافة نظام توليد QR Code
fix(auth): إصلاح مشكلة انتهاء صلاحية التوكن
docs(readme): تحديث خطوات التثبيت
style(dashboard): تحسين تنسيق صفحة الأقسام
```

## 🔀 طلبات السحب (Pull Requests)

### قبل فتح PR

- [ ] تأكد من أن الكود يعمل بشكل صحيح
- [ ] تأكد من اجتياز جميع الاختبارات: `npm test`
- [ ] تأكد من عدم وجود أخطاء في ESLint: `npm run lint`
- [ ] اكتب وصف واضح للتغييرات

### قالب الـ PR

```markdown
## الوصف

وصف مختصر للتغييرات

## نوع التغيير

- [ ] ميزة جديدة
- [ ] إصلاح خطأ
- [ ] تحسين أداء
- [ ] إعادة هيكلة
- [ ] توثيق

## الاختبارات

- [ ] تم إضافة اختبارات جديدة
- [ ] تم تحديث الاختبارات الحالية
- [ ] جميع الاختبارات تمر بنجاح

## لقطات شاشة (إن وجدت)
```

### مراجعة الكود

- يجب أن يحصل كل PR على **موافقة مراجع واحد على الأقل**
- يجب اجتياز جميع فحوصات CI/CD
- يتم دمج الـ PR عبر **Squash and Merge**

## 🐛 الإبلاغ عن الأخطاء

عند الإبلاغ عن خطأ، يرجى تضمين:

1. **وصف الخطأ** — ما المشكلة بالتحديد؟
2. **خطوات إعادة الإنتاج** — كيف يمكن تكرار المشكلة؟
3. **السلوك المتوقع** — ماذا كنت تتوقع أن يحدث؟
4. **السلوك الفعلي** — ماذا حدث بالفعل؟
5. **لقطات شاشة** — إن أمكن
6. **بيئة التشغيل** — نظام التشغيل، المتصفح، إصدار التطبيق

### قالب الإبلاغ عن خطأ

```markdown
## وصف الخطأ

وصف واضح ومختصر للمشكلة

## خطوات إعادة الإنتاج

1. اذهب إلى '...'
2. اضغط على '...'
3. مرر لأسفل إلى '...'
4. شاهد الخطأ

## السلوك المتوقع

ما كنت تتوقع حدوثه

## لقطات شاشة

أضف لقطات شاشة هنا

## البيئة

- النظام: [مثال: Windows 11]
- المتصفح: [مثال: Chrome 120]
- الإصدار: [مثال: 1.0.0]
```

## 💡 اقتراح ميزات جديدة

نرحب باقتراحاتك! يرجى:

1. تأكد من عدم وجود اقتراح مشابه مفتوح

   - ما المشكلة التي تحلها؟
   - كيف تتخيل عملها؟
   - هل لديك تصميم أو مرجع؟
